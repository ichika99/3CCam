## 概述

《明日方舟：终末地》是鹰角网络开发的3D即时策略RPG。游戏采用**四人小队同屏作战**的核心玩法——玩家主控一名角色，其余三名由AI辅助控制，并可随时切换。此外还包含箱庭世界探索和基地建设（集成工业）系统。

相机系统需要同时服务于**多角色同屏的战斗**和**箱庭世界探索及工业建造**——不同场景对相机有不同的要求。

### 相机系统总览

游戏主要包含以下几类相机情景：

> \<!-- 主要把相机分为这几类就可以了：跟随（常规探索、战斗用）、工业建造、特殊相机（包括终结技、剧情演出、策划预设轨道等）。连携技相机不用提，这就是用的战斗相机做了特殊调整 -->

- **跟随相机（探索/战斗共用）**：标准第三人称跟随视角，玩家可完全自由控制旋转。探索和战斗状态共用基本的跟随、避障逻辑，战斗时根据锁定状态有额外调整
- **工业建造相机**：在集成工业系统中使用独立的半俯视自由视角，操控逻辑接近RTS/模拟经营类游戏
- **演出相机**：终结技、连携技、剧情过场等场景使用策划预设的镜头轨道
- **特殊交互相机**：宝箱解锁等特定交互事件触发的聚焦镜头

### 研究价值

- **多角色同屏的构图挑战**：四人同屏作战时，相机如何确保主控角色清晰可见，同时不丢失队友和敌人的信息
- **战斗锁定的双层设计**：相机锁定与战斗目标锁定分离，两套系统各司其职
- **箱庭世界的垂直空间**：地图包含大量垂直结构（矿洞、高台、管道网络），相机碰撞和视角适配面临特殊挑战
- **工业建造视角**：集成工业系统需要一套独立于战斗/探索的相机逻辑

## 跟随相机

探索和战斗状态共用同一套基础跟随相机逻辑，包括跟随、避障、视角操控等。战斗时在此基础上根据锁定状态做额外调整。

### 跟随逻辑

终末地的跟随相机采用标准的第三人称弹簧臂方案：

- 常规世界探索时，角色水平位于**画面中心**
- 在帝江号（飞船）上时，角色位于**画面偏左**位置
- 角色移动时相机有惯性延迟，急停和转向时能感受到相机的"追赶"效果
- 角色静止不动时相机保持稳定，无呼吸或漂浮效果
- 相机仅追踪主控角色，不会因AI队友的移动而调整

::ImageView{src="/images/placeholder/endfield-explore-follow.jpg" caption="探索视角下的跟随距离与角色位置" width="full"}
::

### 视角操控

- 右摇杆/鼠标控制水平+垂直旋转，操控范围自由
- 垂直旋转有上下限位角度限制，防止穿入地面或翻转到角色头顶
- 移动端通过触屏区域的滑动控制视角旋转（左右屏幕区域均可，只要没有控件遮挡）

### 自动回正

相机的自动回正行为因场景而异：

- **常规探索**：角色移动时，如果移动方向与相机朝向不一致，相机会以一定速度**插值回正**到角色后方。当玩家持续输入一个带角度的方向时（如斜向移动），相机会持续处于回正过程中，不会真正回正到角色正后方。角色静止时没有自动回正
- **建造区域**：移动时**没有自动回正**效果，不管角色朝哪个方向走，相机朝向不变，除非玩家主动操纵旋转

### 避障处理

相机的避障逻辑主要包括**障碍物推近**和**遮挡半透**两套机制：

#### 大型障碍物推近

当相机位于墙壁、岩石、大型景观等大型物体后方或内部时，物体会大面积遮挡画面。此时相机会缩短臂长靠近主控角色以实现避障。当角色完全贴近障碍物时，相机也会被迫贴近乃至进入主控角色内部，以确保相机不被障碍物遮挡视野。

如果相机过于接近主控角色导致角色模型遮挡画面，主控角色会切换为 **Dither（抖动点阵）半透效果**，减少遮挡。

碰撞推近有缓动处理，不会产生突然的跳切。离开碰撞区域后恢复原始距离也有过渡过程。

#### 小型障碍物半透

对于电线杆、木桩、工业区建筑、队友角色、NPC 等小型障碍物，相机可以直接穿过。但如果这些障碍物在画面中遮挡了主控角色，大部分障碍物会采用 **Dither 半透效果**处理，确保主控角色始终清晰可见。

经测试，怪物不会触发半透效果，其他大部分小型障碍物都会有半透处理。

## 战斗相机

> \<!-- 这一章中，《多角色同屏构图》《不同敌人类型的相机调整》可以放到同一小节中说明，再根据不同角色类型（主控、队友、敌人等）来进行分析。《闪避与相机》《连携技演出》《战技与镜头反馈》《角色切换》也放到同一小节中说明，都是战斗交互相关的，内容不太多。《终结技演出》可以放到《特殊相机》大章中说明，因为这已经不算常规相机了 -->

战斗相机与探索相机共用基础的跟随和避障逻辑。普通战斗进入/退出时相机参数没有明显的专门调整，是无感切换的。战斗相机的核心区别在于**锁定系统**以及面对不同敌人类型时的参数差异。

### 锁定系统

终末地的锁定系统分为**相机锁定**和**战斗目标锁定**两层，各司其职：

#### 相机锁定

决定**相机望向的敌人**，用怪物身上的锁定点 UI 表示。玩家可主动开启和关闭。

- **关闭时**：相机与常规跟随相机一致，玩家可自由操纵视角，没有额外调整
- **开启时**：相机望向锁定目标，玩家无法自由旋转视角。原先的视角操纵输入变为**切换相机锁定目标**。相机对怪物的锁定是比较弱的——第一优先级是保证主控角色在屏幕中心，其次才是调整朝向以确保锁定目标在屏幕相对居中的位置。相机焦点仍然在角色上，或是角色与怪物连线上非常靠近角色的某一点

#### 战斗目标锁定

决定**角色攻击时朝向的目标**，用怪物血条下方的小箭头表示。进入战斗时默认开启。

- 当战斗目标距离角色较近时，角色攻击会**自动朝向目标**
- 当战斗目标距离较远时，不会进行朝向修正
- **目标选择规则**：如果玩家启用了相机锁定，则战斗目标即为相机锁定目标；如果未启用相机锁定，则战斗目标由内置算法决定，体感上为每隔一定时间更新为离玩家较近的怪物

### 多角色同屏构图

四名角色同时出现在战场上，相机的处理策略：

**主控角色优先**：
- 相机始终以当前主控角色为跟踪目标，主控角色保持在屏幕水平中心位置
- 不会因队友移动而偏移构图

**队友可见性处理**：
- AI 控制的队友会**自动调整站位**以避免遮挡主控角色的视线——通过 AI 行为层面解决遮挡问题，而非相机适配
- 在群战中，相机**不会主动拉远到容纳所有人的距离**，而是保持以主控角色为核心的视角

**敌人信息传达**：
- 依赖 UI 元素（伤害数字、状态图标、Boss 血条等）补偿相机视野之外的信息
- 非锁定状态下也能通过 UI 一定程度掌握敌人状态；开启锁定则会尽量确保锁定目标可见
- 受击时屏幕中央会出现**环形伤害方向指示器**，以角色为圆心指示受击方向

### 不同敌人类型的相机调整

面对不同类型的敌人，战斗相机会有不同的参数调整：

- 面对大型 Boss 时，相机会**自动拉远**更多距离，确保 Boss 的关键攻击前摇和 AOE 范围指示器在视野内
- Boss 蓄力攻击阶段，地面出现红色/橙色范围指示器，相机高度偏移增加以确保指示器清晰可见
- Boss 相位转换（切换攻击模式、召唤小怪等）通常伴随短暂的镜头演出
- 面对普通小怪时，相机参数基本无特殊变化

### 闪避与相机

闪避时相机无特殊表现——继续正常跟随角色移动，不触发慢动作、zoom 或其他特殊镜头效果。完美闪避有短暂时停和无敌帧，但不改变相机行为。

### 终结技演出

终结技（大招）是战斗中最具视觉冲击力的镜头时刻：

- 释放终结技时进入**短暂的演出模式**，时间暂停或极度减缓，相机切换到预设的演出轨道
- 每个角色的终结技有**独立设计的镜头编排**：特写推入、动态环绕、低角度仰拍等
- 演出期间角色有专属的全屏动画表现，背景适当虚化以突出角色
- 演出结束后相机快速平滑过渡切回战斗视角

::

### 连携技演出

连携技的镜头演出比终结技更为克制：

- 触发时有**短暂的镜头特写**，快速 zoom-in 到释放连携技的角色
- 不完全暂停时间，而是短暂的慢动作（hit-stop 效果）
- 演出被刻意设计得简短以避免打断战斗流
- 多个连携技连续触发时形成快节奏的连续特写

### 战技与镜头反馈

角色释放战技（主动技能）时基本不影响相机行为，没有专门的相机配置。但可以通过配置**震屏（camera shake）**等效果实现简易的镜头反馈，为技能增加打击感。

整体而言，终末地的镜头反馈偏克制——普攻和技能命中时的 shake 幅度较轻，连携技触发时有顿帧配合轻度 shake 和 zoom-in。

### 角色切换

战斗中切换主控角色时，游戏的做法是**将目标角色与当前角色交换位置**，而非将相机移动到目标角色处。因此相机本身只会因两个角色的身高差产生轻微的竖直位移，不存在大幅度的镜头移动。

## 工业建造相机

这是终末地较为独有的相机模式，在进入集成工业系统的建造/编辑界面时生效：

- 相机从角色跟随视角切换为**半俯视的自由视角**，可自由平移、旋转和缩放
- 缩放范围较大，从近距离细节查看到较远距离的全局规划俯瞰均支持
- 此模式下相机操控逻辑更接近 RTS/模拟经营类游戏，与战斗/探索的 TPS 操控形成明显区分
- 退出建造界面时，相机通过一次平滑过渡回到角色跟随视角

## 特殊相机

### 过场演出

剧情过场由策划针对每个场景手动配置镜头，表现力取决于策划个人能力。整体采用实时渲染方案：

- 剧情对话使用预设的对话构图（正反打、中景等）
- 重要剧情节点使用更精心编排的镜头语言——推拉、摇移、跟拍等
- 过场与游玩之间存在明确的镜头切换/黑屏过渡，非一镜到底

::ImageView{src="/images/placeholder/endfield-cutscene.jpg" caption="剧情过场的实时渲染演出" width="full"}
::

### 交互聚焦镜头

游戏中存在一些同质化的特殊交互镜头设计。例如有一种宝箱需要玩家在周围找到三个收集物才能解锁，收集完毕后会**黑幕过渡**，切换到聚焦于宝箱的特殊相机并播放解锁动画。

这种设计的优点是能稳定聚焦到交互目标，让玩家清晰看到解锁结果；缺点是需要一个黑幕切换相机的过程，打断了探索的连贯感。一种可能的优化方案是在不黑幕的情况下，让跟随相机通过平滑过渡自然转向聚焦目标，但这对相机过渡的控制精度要求更高。

## 设计分析与总结

### 设计取舍

终末地的相机设计反映了一系列有意识的取舍：

**策略体验优先**：
- 镜头反馈偏克制，camera shake 和演出镜头的使用较为克制，避免画面"过于吵闹"
- 闪避不触发特殊镜头效果，不鼓励玩家追求精确操作奖励
- 战斗反馈的高光集中在连携技和终结技的演出上

**信息传达的分层策略**：
- 相机负责提供主控角色周边的空间信息
- 更全局的信息（队友状态、敌人位置、受击方向）交给 UI 系统处理
- 这种"相机 + UI"协作的模式是多角色同屏战斗的务实选择

### 优点

- **AI 行为解决相机遮挡**：队友自动调整站位避免遮挡主控角色，从行为层面解决问题，不影响玩家对相机的控制预期
- **双层锁定系统设计**：相机锁定与战斗目标锁定分离，相机锁定保持弱锁定（优先角色居中），战斗目标锁定独立处理朝向，两者可独立开关
- 探索/战斗/建造三种模式共用基础逻辑、各有特化，模式间过渡平滑
- 终结技/连携技的演出镜头时长控制得当，既有视觉冲击力又不过度打断战斗流
- Dither 半透处理有效解决了小型障碍物和角色遮挡主控视野的问题

### 缺点

- 非锁定状态下面对多目标时精确控制感较弱
- 闪避时缺乏相机反馈，完美闪避的成就感薄弱
- 角色切换采用交换位置的方式虽然对相机友好，但在视觉上可能造成一定的"不自然"感
- 大型 Boss 战中相机拉远幅度有时不够充分

### 值得借鉴

1. **AI 行为解决相机遮挡**：多角色同屏时让 AI 角色主动调整站位，比调整相机位置更优雅
2. **相机 + UI 协作传达信息**：承认相机无法在多角色场景中展示所有信息，将全局信息交给 UI（环形伤害方向指示器、状态图标等），务实且有效
3. **建造模式的视角分离**：为工业建造提供独立的半俯视自由视角，而非强迫 TPS 视角完成建造操作
4. **Dither 半透的障碍物处理**：区分大型/小型障碍物采用不同策略——大型推近、小型穿透+半透——是灵活的组合方案
5. **双层锁定的解耦设计**：将"相机望向谁"和"攻击打向谁"拆分为两套独立系统，给予玩家更灵活的控制粒度

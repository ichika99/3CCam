   ## 概述

   《明日方舟：终末地》是鹰角网络开发的3D即时策略RPG。游戏采用**四人小队同屏作战**的核心玩法——玩家主控一名角色，其余三名由AI辅助控制，并可随时切换。此外还包含箱庭世界探索和基地建设（集成工业）系统。

   相机系统需要同时服务于**多角色同屏的战斗**和**箱庭世界探索及工业建造**——不同场景对相机有不同的要求。

   ### 相机系统总览

   游戏主要包含以下几类相机情景：

   - **跟随相机**：包括玩家自由探索时和战斗锁定时两种相机，采用第三人称跟随视角。探索和战斗状态共用基本的跟随、避障逻辑，战斗时根据锁定状态有额外调整
   - **工业建造相机**：在集成工业系统中使用独立的俯视角相机，操控逻辑接近RTS/模拟经营类游戏
   - **特殊相机**：终结技演出、剧情过场等场景使用策划预设的镜头轨道或特殊相机逻辑

   ### 研究价值

   - **多角色同屏的构图挑战**：四人同屏作战时，相机如何确保主控角色清晰可见，同时不丢失队友和敌人的信息
   - **战斗锁定的双层设计**：相机锁定与战斗目标锁定分离，两套系统各司其职
   - **箱庭世界的垂直空间**：地图包含大量垂直结构（矿洞、高台、管道网络），相机碰撞和视角适配面临特殊挑战
   - **工业建造视角**：集成工业系统需要一套独立于战斗/探索的相机逻辑

   ## 跟随相机

   探索和战斗状态共用同一套基础跟随相机逻辑，包括跟随、避障、视角操控等。战斗时在此基础上根据锁定状态做额外调整。

   ### 跟随逻辑

   终末地的跟随相机采用标准的第三人称弹簧臂方案：

   - 常规世界探索时，角色水平位于**画面中心**
   - 在帝江号（飞船）上时，角色位于**画面偏左**位置
   - 角色移动时相机有惯性延迟，急停和转向时能感受到相机的"追赶"效果
   - 角色静止不动时相机保持稳定，无呼吸或漂浮效果
   - 相机仅追踪主控角色，不会因AI队友的移动而调整

   ### 视角操控

   - 右摇杆/鼠标/触屏控制水平+垂直旋转，操控范围自由
   - 垂直旋转有上下限位角度限制，防止穿入地面或翻转到角色头顶

   ### 自动回正

   相机的自动回正行为因场景而异：

   - **常规探索**：角色移动时，如果移动方向与相机朝向不一致，相机会以一定速度**插值回正**到角色后方。当玩家持续输入一个带角度的方向时（如斜向移动），相机会持续处于回正过程中，不会真正回正到角色正后方。角色静止时没有自动回正

   ::VideoEmbed{src="\media\探索移动时镜头往运动方向回正.mp4" caption="常规移动时相机慢慢转向角色后方"}
::

   - **建造区域**：移动时**没有自动回正**效果，不管角色朝哪个方向走，相机朝向不变，除非玩家主动操纵旋转

   ::VideoEmbed{src="\media\建造区移动时镜头不回正.mp4" caption="工业区建造模式下移动时相机不会自行转动"}
::

   ### 避障处理

   相机的避障逻辑主要包括**障碍物推近**和**遮挡半透**两套机制：

   #### 大型障碍物推近

   当相机位于墙壁、岩石、大型景观等大型物体后方或内部时，物体会大面积遮挡画面。此时相机会缩短臂长靠近主控角色以实现避障。当角色完全贴近障碍物时，相机也会被迫贴近乃至进入主控角色内部，以确保相机不被障碍物遮挡视野。

   如果相机过于接近主控角色导致角色模型遮挡画面，主控角色会切换为 **Dither（抖动点阵）半透效果**，减少遮挡。

   ::VideoEmbed{src="\media\避障——相机撞墙.mp4" caption="撞墙时相机避障"}
::

   碰撞推近有缓动处理，不会产生突然的跳切。离开碰撞区域后恢复原始距离也有过渡过程。

   #### 小型障碍物半透

   对于电线杆、木桩、工业区建筑、队友角色、NPC 等小型障碍物，相机可以直接穿过。但如果这些障碍物在画面中遮挡了主控角色，大部分障碍物会采用 **Dither 半透效果**处理，确保主控角色始终清晰可见。

   怪物不会触发半透效果，其他大部分小型障碍物都会有半透处理。

   ::VideoEmbed{src="\media\避障——镜头有阻挡的dither半透.mp4" caption="小型障碍物遮挡视线时半透处理"}
::

   ## 战斗相机

   战斗相机与探索相机共用基础的跟随和避障逻辑，相机的基本参数也与常规跟随状态一致，从自由探索到进入战斗是无感切换的。战斗相机的核心区别在于**锁定系统**以及一些战斗、角色交互行为。

   ### 锁定系统

   终末地的锁定系统分为**相机锁定**和**战斗目标锁定**两层，各司其职：

   ::ImageView{src="\images\placeholder\锁定UI.png" caption="两种锁定对应的UI"}
::

   #### 相机锁定

   决定**相机望向的敌人**，用怪物身上的锁定点 UI 表示。玩家可主动开启和关闭。

   - **关闭时**：相机与常规跟随相机一致，玩家可自由操纵视角，没有额外调整
   - **开启时**：相机望向锁定目标，玩家无法自由旋转视角。原先的视角操纵输入变为**切换相机锁定目标**。相机对怪物的锁定是比较弱的——第一优先级是保证主控角色在屏幕中心，其次才是调整朝向以使锁定目标在屏幕中相对中间的位置。相机焦点仍然在角色上，或是角色与怪物连线上非常靠近角色的某一点

   #### 战斗目标锁定

   决定**角色攻击时朝向的目标**，用怪物血条下方的小箭头表示。目标锁定系统进入战斗时默认开启，但UI显示跟随血条，满血时不显示锁定UI。

   - 当战斗目标距离角色较近时，角色攻击会**自动朝向目标**
   - 当战斗目标距离较远时，不会进行朝向修正
   - **目标选择规则**：如果玩家启用了相机锁定，则战斗目标即为相机锁定目标；如果未启用相机锁定，则战斗目标由内置算法决定，体感上为每隔一定时间更新为离玩家较近的怪物，或是玩家攻击命中的怪物

   ::VideoEmbed{src="\media\锁定表现.mp4" caption="相机锁定与战斗目标锁定的表现"}
::

   ### 战斗构图与多角色处理

   四人同屏作战时，相机需要处理主控角色、队友、敌人三类对象的信息呈现：

   **主控角色**：

   - 相机始终以当前主控角色为跟踪目标，主控角色保持在屏幕水平中心位置
   - 不会因队友或敌人移动而偏移构图

   **队友**：

   - AI 控制的队友会**自动调整站位**以避免遮挡主控角色的视线——通过 AI 行为层面解决遮挡问题，而非相机适配
   - 在群战中，相机**不会主动拉远到容纳所有人的距离**，而是保持以主控角色为核心的视角

   **敌人**：

   - 大多数敌人战斗相机参数都是一致的默认参数，基本没有特殊调整
   - 通过各种预警特效来提示玩家即将到来的攻击行为
   - 依赖 UI 元素（伤害数字、状态图标、Boss 血条等）补偿相机视野之外的信息
   - 受击时屏幕中央会出现**环形伤害方向指示器**，以角色为圆心指示受击方向

   ::VideoEmbed{src="\media\战斗演示.mp4" caption="战斗演示"}

   ### 战斗交互与镜头反馈

   战斗中的各类交互操作对相机的影响整体偏克制：

   - **闪避**：相机无特殊表现，完美闪避有短暂时停和无敌帧，不改变相机行为
   - **战技**：基本不影响相机行为，没有专门的相机配置
   - **连携技**：触发时有**短暂的镜头特写**，快速 zoom-in 到释放连携技的角色，伴随短暂的 hit-stop 效果。演出被刻意设计得简短以避免打断战斗流，多个连携技连续触发时形成快节奏的连续特写
   ::VideoEmbed{src="\media\连携技.mp4" caption="连携技有简易的zoom in效果"}
::
   - **角色切换**：游戏的做法是**将目标角色与当前角色交换位置**，而非将相机移动到目标角色处。因此相机本身只会因两个角色的身高差产生轻微的竖直位移，不存在大幅度的镜头移动
   - **终结技**：不同角色采用定制的sequence来进行演出
   
   整体而言，终末地的镜头反馈偏克制——普攻和技能命中时的 shake 幅度较轻，连携技触发时有顿帧配合轻度 shake 和 zoom-in。战斗反馈的高光集中在终结技的演出上。

   ## 工业建造相机
   
   这是终末地较为独有的相机模式，在进入集成工业系统的建造/编辑界面时生效：
   
   - 相机从角色跟随视角切换为**半俯视的自由视角**，可自由平移、旋转和缩放
   - 缩放范围较大，从近距离细节查看到较远距离的全局规划俯瞰均支持
   - 此模式下相机操控逻辑更接近 RTS/模拟经营类游戏，与战斗/探索的 TPS 操控形成明显区分
   - 退出建造界面时，通过后处理的相机特效切回常规跟随相机

   ## 特殊相机
   
   ### 终结技演出
   
   终结技（大招）是战斗中最具视觉冲击力的镜头时刻：
   
   - 释放终结技时进入**短暂的演出模式**，时间暂停或极度减缓，相机切换到预设的演出轨道
   - 每个角色的终结技有**独立设计的镜头编排**：特写推入、动态环绕、低角度仰拍等
   - 演出期间角色有专属的全屏动画表现，有角色专属的背景遮罩
   - 演出结束后切回原相机，部分角色有过渡效果
   
   ::VideoEmbed{src="\media\终结技.mp4" caption="终结技的特殊镜头，莱万汀开大结束时有个快速切回常规镜头的过渡"}
::
   
   ### 过场演出
   
   主要包括剧情过场的演出，世界探索中的部分特殊场景，Boss的某些特殊招式等，通常又策划配置固定运动轨迹的镜头
   
   ### 固定交互镜头

   游戏中存在一些同质化的特殊交互镜头设计。例如有一种宝箱需要玩家在周围找到三个收集物才能解锁，收集完毕后会**黑幕过渡**，切换到聚焦于宝箱的特殊相机并播放解锁动画。

   这种设计的优点是能稳定聚焦到交互目标，让玩家清晰看到解锁结果；缺点是需要一个黑幕切换相机的过程，打断了探索的连贯感。一种可能的优化方案是在不黑幕的情况下，让跟随相机通过平滑过渡自然转向聚焦目标，但这对相机过渡的控制精度要求更高。

   ## 设计分析与总结

   ### 设计取舍

   终末地的相机设计反映了一系列有意识的取舍：

   **策略体验优先**：

   - 镜头反馈偏克制，camera shake 和演出镜头的使用较为克制，避免画面"过于吵闹"
   - 探索、战斗锁定等相机FOV、焦距、臂长等基本只使用同一套参数，游戏的探索、战斗状态相机无感切换
   - 战斗反馈的高光集中在终结技的演出和连续的连携技聚焦上
   
   **信息传达的分层策略**：

   - 相机负责提供主控角色周边的空间信息
   - 更全局的信息（队友状态、敌人位置、受击方向）交给 UI 系统处理
   - 这种"相机 + UI"协作的模式是多角色同屏战斗的务实选择
   
   ### 优点

   - **AI 行为解决相机遮挡**：队友自动调整站位避免遮挡主控角色，从行为层面解决问题，不影响玩家对相机的控制预期
   - **双层锁定系统设计**：相机锁定与战斗目标锁定分离，相机锁定保持弱锁定（优先角色居中），战斗目标锁定独立处理角色朝向，两者弱关联
   - 探索/战斗/建造三种模式共用基础逻辑、各有特化，模式间过渡平滑
   - 终结技/连携技的演出镜头时长控制得当，既有视觉冲击力又不过度打断战斗流
   - Dither 半透处理有效解决了小型障碍物和角色遮挡主控视野的问题，整体相机避障效果优秀
   
   ### 缺点

   - 非锁定状态下面对多目标时精确控制感较弱
   - 锁定状态感知比较弱，对于灵活的怪物或群战跟踪性能不太够
   
   ### 值得借鉴

   1. **AI 行为解决相机遮挡**：多角色同屏时让 AI 角色主动调整站位，比调整相机位置更优雅
   2. **相机 + UI 协作传达信息**：承认相机无法在多角色场景中展示所有信息，将全局信息交给 UI（环形伤害方向指示器、状态图标等），务实且有效
   3. **建造模式的视角分离**：为工业建造提供独立的半俯视自由视角，而非强迫 TPS 视角完成建造操作
   4. **Dither 半透的障碍物处理**：区分大型/小型障碍物采用不同策略——大型推近、小型穿透+半透——是灵活的组合方案
   5. **双层锁定的解耦设计**：将"相机望向谁"和"攻击打向谁"拆分为两套独立系统，给予玩家更灵活的控制粒度
